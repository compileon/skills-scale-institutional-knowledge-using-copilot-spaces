# Role Responsibility Matrix (example)

This lightweight matrix helps teams quickly map ownership for project artifacts and lifecycle activities.

Suggested columns: Artifact / Activity | Primary (R) | Accountable (A) | Consulted (C) | Informed (I)

Examples:
- Project One-pager | PdM (R) | Sponsor (A) | PM, UX (C) | Team (I)
- Backlog prioritization | PdM (R) | PdM (A) | PM, BA, Devs (C) | Stakeholders (I)
- Sprint planning | Devs (R) | PM / Scrum Master (A) | PdM, QA (C) | Stakeholders (I)
- Release sign-off | QA (R) | PM (A) | Devs, Support (C) | Stakeholders (I)

How to use:
- Add a short matrix for each project in the project README or one-pager.
- Update as team composition changes.
